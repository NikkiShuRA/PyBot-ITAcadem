# Как контрибьютить

Основные правила разработки описаны в `CONTRIBUTING.md`, `AGENTS.md` и `ARCHITECTURE.md`.

## Workflow

1. Работайте в feature/fix-ветке.
2. Не вносите архитектурные изменения без проверки ADR и архитектурных инвариантов.
3. Держите handlers тонкими, а бизнес-логику в сервисах.
4. Обновляйте тесты и документацию вместе с изменением поведения.

## Локальная проверка

```bash
just quality-gate
just docs-build
```

## Для PR

- опишите изменение поведения;
- перечислите измененные файлы и ключевые решения;
- укажите результаты проверок;
- отдельно отметьте то, что не удалось проверить локально.
