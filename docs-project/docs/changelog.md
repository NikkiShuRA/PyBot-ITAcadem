# Changelog

Источником истории изменений остается корневой `CHANGELOG.md` репозитория.

Эта страница существует, чтобы навигация сайта была полной и не ломала сборку.

## Как использовать changelog в проекте

- фиксируйте заметные user-facing изменения;
- не забывайте про миграции и deployment impact;
- для архитектурных решений используйте ADR, а не changelog.
